Compilador:
P:LOs complidarres son programas  los cuales tienen la capacidad de transformar a binario las instruciones dadas por el lenguaje de programacion y asi poder completar la comunicacion entre el lenguajee y la computadora
F: Un compilador es, a grandes rasgos, un traductor. Es el encargado de transformar el código fuente del programa que hemos creado (generalmente usando un lenguaje de alto nivel), junto con todas las dependencias y librerías necesarias para que el programa pueda ejecutarse y funcionar sin problemas, en un archivo binario. Este compilador se encarga de comprobar que no hay errores en el código (para evitar fallos críticos), así como de analizar y agrupar la sintaxis y la estructura del programa. Si todo está correcto, lo pasa a un lenguaje intermedio común, ensamblador, para, posteriormente, optimizarlo y traducirlo a lenguaje binario. De esta manera, a través del sistema operativo, el programa se puede ejecutar en la máquina. (Velasco, R. (2021, 1 junio). Sin él no existirían los programas: qué es un compilador. SoftZone. https://www.softzone.es/programas/lenguajes/que-es-compilador/)
Ciclo de Desarrollo de Software
P:
F: El ciclo de vida de un software es una secuencia estructurada y bien definida de las etapas en ingeniería de software para desarrollar el software deseado. (Solbyte. (2022, 25 abril). Ciclo de vida de un software: qué es, modelo, etapas | Solbyte. Blog Solbyte. https://www.solbyte.com/blog/ciclo-de-vida-del-software/)
este ciclo se contiene de 
1. Planificacion:
P:Durante esta fase del desarrollo mediante la recolecta de informacion se comianza a estructurar y planificar el plan de desarrollo del programa siendo que esta fase se consideran  cosas tales como
F: En esta fase se incluyen tareas como la determinación del ámbito del proyecto, un estudio de viabilidad, análisis de riesgos, costes estimados, asignación de recursos en las distintas etapas, etc. (Solbyte. (2022, 25 abril). Ciclo de vida de un software: qué es, modelo, etapas | Solbyte. Blog Solbyte. https://www.solbyte.com/blog/ciclo-de-vida-del-software/)
2. Analisis:
P:
F: Revisión de los requisitos componiéndolos como un todo coherente, revisando si existen carencias o inconsistencias. (Ciclo de vida en el Desarrollo de Software, primera parte 2022. (2021, 28 octubre). IEDGE Business School. https://www.iedge.eu/pablo-almunia-ciclo-de-vida-en-el-desarrollo-de-software-primera-parte)
3. Diseño:
P:
F: Los desarrolladores y arquitectos de software comienzan a diseñar el software después de haber identificado los requisitos. A menudo, la creación rápida de prototipos también se incluye en el proceso de diseño. Esto genera una idea más clara del resultado final desde el punto de vista del cliente final, y permite incluir importantes cuestiones como la experiencia de cliente, usabilidad, etcétera. Este enfoque requiere una comparación de las soluciones que existen en el mercado a través de estudios y análisis detallados. Al final de esta fase, los equipos obtienen documentos de diseño con los patrones y componentes elegidos para el proyecto, y el código utilizado como punto de partida para una ampliación de las necesidades de desarrollo. (TRPlane. (2021, 16 octubre). Comprender el ciclo de Cida de Desarrollo de Software (SLDC). TRPlane.com. https://www.trplane.com/comprender-el-ciclo-de-cida-de-desarrollo-de-software-sldc/)
4. Inplementacion/ Codificacion:
P:
F: Implementación de las diferentes funcionalidades por medio de lenguajes de programación o parametrización de paquetes o sistemas preexistentes (Ciclo de vida en el Desarrollo de Software, primera parte 2022. (2021, 28 octubre). IEDGE Business School. https://www.iedge.eu/pablo-almunia-ciclo-de-vida-en-el-desarrollo-de-software-primera-parte)
5. Testing:
P:
F: Se estima que el 50% de todos los procesos de desarrollo de software deberían ser evaluados. Los errores pueden arruinar el software tanto a nivel crítico y hasta el punto de ser eliminado. Las pruebas de Software se hacen mientras se codifica y suelen hacerlo los desarrolladores y otros expertos evaluadores a varios niveles. Esto incluye evaluación de módulos, evaluación del programa, evaluación del producto, evaluación interna y finalmente evaluación con el consumidor final. Encontrar errores y su remedio a tiempo es la llave para conseguir un software fiable. (Software - ciclo de vida de desarrollo. (s. f.). https://www.tutorialspoint.com/es/software_engineering/software_development_life_cycle.htm)
6. Integracion:
P:
F: El Software puede necesitar estar integrado con las bibliotecas, Bases de datos o con otro u otros programas. Esta fase del SDLC se focaliza en la integración del software con las entidades del mundo exterior. (Software - ciclo de vida de desarrollo. (s. f.). https://www.tutorialspoint.com/es/software_engineering/software_development_life_cycle.htm)
7. Matenimiento
P:
F: Cuando el software pasa por todas las etapas sin complicaciones, se realiza un ciclo de mantenimiento en el que se actualiza y modifica cada cierto tiempo para adaptarse a los cambios y realizar mejoras sobre el mismo. (TRPlane. (2021, 16 octubre). Comprender el ciclo de Cida de Desarrollo de Software (SLDC). TRPlane.com. https://www.trplane.com/comprender-el-ciclo-de-cida-de-desarrollo-de-software-sldc/) 
Especificacion de requisitos de Software
P: Estos son lass expectativas, diseño, estandares, requistos de los usuarios y su finalidad como producto
F: La especificación de requisitos de software (ERS) es una descripción completa del comportamiento del sistema que se va a desarrollar. Incluye un conjunto de casos de uso que describe todas las interacciones que tendrán los usuarios con el software. Los casos de uso también son conocidos como requisitos funcionales.
